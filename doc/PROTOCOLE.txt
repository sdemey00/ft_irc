PROTOCOL IRC - reference : RFC 1459:

Text base message line:
:[prefix] COMMAND [parameters] [message/trailing]

COMMAND:
Operator:
	KICK - Eject a client from the channel
	INVITE - Invite a client to a channel
	TOPIC - Change or view the channel topic
	MODE - Change the channel's mode:
		* i: Set/remove Invite-ony channel
		* t: Set/remove the restrictions of the TOPIC command to channel operators
		* k: Set/remove the channel key (password)
		* o: Dive/take channel operator privilege
		* l: Set/remove the user limit to channel
		(example: MODE #canal +i || MODE user +o) -> channel mode AND user mode
Other:
	PASS - password
	NICK new_pseudo - Change user's pseudonyme
	USER ident mode full_name - Send user ident informations to server
	JOIN #channel - Join a channel
	PART #channel - Leave a channel
	PRIVMSG #name :message - send a private text to a user or a channel
	NOTICE - similar to PRIVMSG (auto replies?)
	QUIT - Disconnect from the server
(some command need a confirmation message like " :user!ident@host JOIN #channel ")

ERRORS/REPLIES:
Response and code state:
	replies :
	RPL_WELCOME (001): welcome message when connexion success
	RPL_TOPIC (332): channel subject
	errors :
	ERR_NICKNAMEINUSE (433): pseudo already in use (NICK answer)
	ERR_NEEDMOREPARAMS (461): Error when a command need more params
	ERR_UNKNOWNCOMMAND (421): Error when uknown command
	...
Example:
	:server_name 433 * new_pseudo : Nickname is already in use

MOTD = Message of the Day: at connexion to an IRC server, MOTD is sent (RPL_WELCOME)
PING PONG = protocol to maintain sustanable connectivity betwen server/client
	Server send a message to test connectivity
	PING :server_name
	Client or Server respond to conform
	:server_name PONG
